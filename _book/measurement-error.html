5  Measurement Error

5.1 Binary outcome

Measurement error in a binary outcome can be addressed when having estimates of sensitivity and specificity. The easiest correction method is for non-differential error, which is also assumed to be independent of other measurement errors.

(Carroll et al. 2006) (pp. 346-347) describe a correction method via maximum likelihood or Bayesian methods using the following equations:

In the following formulas, \(\mathbf{Y}\) is the true binary outcome with \(\mathbf{Y}=1\) meaning that a participant truely has the outcome, while \(\mathbf{Y}=0\) means that the individual does truely note have the outcome. In contrast, \(\mathbf{S}\) is the measured and potentially misclassified binary outcome, with \(\mathbf{S}=1\) meaning that, based on the measurement, the participant has the outcome, while \(\mathbf{S}=0\) means that, based on the measurement, the participant does not have the outcome. In addition, \(\mathbf{Z}\) is the set of predictor variables.

Measurement error is parameterised by sensitivity (\(\pi_1\)) and specificity (\(\pi_0\)), which are defined as follows:

\[ \begin{align} \pi_1 &= pr(\mathbf{S}=1\,|\,\mathbf{Y}=1,\,\mathbf{Z})\\ \pi_0 &= pr(\mathbf{S}=0\,|\,\mathbf{Y}=0,\,\mathbf{Z}) \end{align} \]

If we had the true binary outcome, we would use a logistic model for analysis, described as follow:

\[ pr(\mathbf{Y}=1\,|\,\mathbf{Z})=H(\beta_0+\beta_z^t\,\mathbf{Z}) \]

However, when using \(\mathbf{S}\) instead of \(\mathbf{Y}\), the probability model needs to be adjusted as follows:

\[ pr(\mathbf{S}=1\,|\,\mathbf{Z})=(1-\pi_0)+(\pi_1+\pi_0-1)\,H(\beta_0+\beta_z^t\,\mathbf{Z}) \]

The authors describe the loglikelihood function to be maximised as:

\[ \sum_{i=1}^n[\mathbf{S}_i\,log\{ \Psi(\mathbf{S}, \mathbf{Z}, \pi_0, \pi_1, \beta_0, \beta_z)\}+(1-\mathbf{S}_i)\,log\{1-\Psi(\mathbf{S},\mathbf{Z},\pi_0,\pi_1,\beta_0,\beta_z)\}] \]

with \(\Psi(\mathbf{S},\mathbf{Z},\pi_0,\pi_1,\beta_0,\beta_z)\) equal to \(pr(\mathbf{S}=1\,|\,\mathbf{Z})\). Let’s therefore simplify to:

\[ \sum_{i=1}^n[\mathbf{S}_i\,log\{pr(\mathbf{S}=1\,|\,\mathbf{Z})\}+(1-\mathbf{S}_i)\,log\{1-pr(\mathbf{S}=1\,|\,\mathbf{Z})\}] \]

5.1.1 Maximum likelihood

One approach of conducting the correction in R is to use optim() to maximise the likelihood, or, to be more precise, to minimise the negative loglikelihood.

First, a function needs to be created that is calculating the negative loglikelihood (nll) based on parameter values (argument pars), data (argument data), and additional inputs, in our case sensitivity (argument se) and specificity (argument sp). The following code assumes a single predictor x.

nll <- function(pars, data, se, sp) {
    # Parameters
    a <- pars[1]
    b <- pars[2]

    # Calculate p
    p <- (1 - sp) + (se + sp - 1) * plogis(a + b * data$x)

    # Calculate likelihood
    ll <- data$yObs * log(p) + (1 - data$yObs) * log(1 - p)

    # Return
    -sum(ll)
  }

First, nll extracts the two parameters a and b from the argument pars. a and b are the regression coefficients that we want to estimate. Next, p is calculated, which coresponds to \(pr(\mathbf{S}=1\,|\,\mathbf{Z})\). plogis() transforms the linear predictor on the log-odds scale to a probability. Next, the likelihood for each observation is calculated based on the formula above, with yObs, the observed outcome of a certain individual, refering to \(\mathbf{S}_i\). ll is a vector that contains the contribution to the total likelihood from each observation. Finally, -sum(ll) calculates the total negative loglikelihood that is then minimised by the optim() function. Again, the negative loglikelihood is used because optim() does minimisation instead of maximisation and minimising the negative loglikelihood is similar to maximising the (positive) loglikelihood.

mle <- optim(
  par = c(a = -3, b = 0), fn = nll,
  data = df, se = 0.72, sp = 0.85,
  hessian = TRUE
)
  
std_error <- mle$hessian %>%
  solve() %>%
  diag() %>%
  sqrt()

When using optim(), we need to provide a named vector of starting values for the parameters in argument par, the nll function in argument fn, the data in argument data, as well as values for the additional arguments of function nll, in our case values for sensitivity and specificity. We also set hessian = TRUE, because we use it to calculate the parameters’ standard errors.

5.1.2 Bayesian approach

One problem with the maximum likelihood approach is that we can only provide a point estimate of sensitivity and specificity when calling optim() and therefore cannot include any uncertainty. However, a Bayesian approach enables us to do so. The basic idea is that we include sensitivity and specificity as parameters in the Bayesian model and provide informative priors that correspond to our uncertainty. Since there is no information to update our priors for sensitivity and specificity, their posteriors are identical to their priors and the final samples reflect this uncertainty as well.

data {
  int<lower=0> N; // Number of observations
  array[N] int<lower=0, upper=1> y_obs; // Observed outcome
  array[N] int<lower=0, upper=1> x; // Binary predictor
}

parameters {
  real a; // Intercept (logodds scale)
  real b; // Coefficient (logodds scale)
  real<lower=0, upper=1> se; // Sensitivity
  real<lower=0, upper=1> sp; // Specificity
}

model {
  // Priors
  a ~ normal(0, 2.5); // Weakly informative
  b ~ normal(0, 2.5); // Weakly informative
  se ~ beta(221.6, 85.61); // Highly informative
  sp ~ beta(1037, 182.1); // Highly informative
  
  // Likelihood
  for (n in 1:N) {
    y_obs[n] ~ bernoulli((1 - sp) + (se + sp - 1) * inv_logit(a + b * x[n]));
  }
}

generated quantities {
  real oddsratio = exp(b);
}

The above Stan code similarly assumes a single predictor x. As described above, sensitivity se and specificity sp are included as parameters instead of data. In the example above, priors for regression parameters a and b are only weakly informative, but more informative priors are possible. From previous simulations we know that more informative priors might both improve precision or additionally bias the results. A bad prior for the intercept a might bias the estimate for b even if b’s prior corresponds to the true value. Therefore, conduct detailled simulations when applying any measurement error correction. Priors of sensitivity and specificity correspond encode the 95% confidence interval of their estimated values. In the example above, a meta-analysis reports estimates for sensitivity and specificity with 95% confidence intervals of 0.67-0.77 and 0.83-0.87, respectively. As priors, the beta distributions, for which the 0.025 and 0.975 quantiles fall together with the lower and upper bound of the confidence interval, are selected. One possibility to find the corresponding distributions is the Probability Distribution Explorer. In the likelihood of the Stan model, the probability model for \(pr(\mathbf{S}=1\,|\,\mathbf{Z})\) is used. As the result is a probability, distribution bernoulli instead of bernoulli_logit must be used. Function inv_logit() tranforms log-odds to probabilities, similar to plogis() above. Finally, the odds ratio is calculated in the generated quantities block.

# Data-generating process (dgp)
sim_dgp <- function(n, pX, pAlpha, ORx) {
  # Draw a single value for sensitivity and specificity
  se <- rbeta(1, 221.6, 85.61) %>% round(3)
  sp <- rbeta(1, 1037, 182.1) %>% round(3)

  dags <- list(sim_mod)
  args <- list(
    pX = pX,
    pAlpha = pAlpha,
    ORx = ORx,
    se = se,
    sp = sp
  )

  df <- run_sim(
    dags = dags,
    args = args,
    n = n
  )

  out <- list(
    df = df
  )

  return(out)
}

# Analysis functions, i.e., method functions
sim_method_corrected <- function(df) {
  # Load Stan model that corrects and has weakly informative priors for a and b
  model <- cmdstan_model("Stan/outc_miscl.stan")

  dat <- list(
    N = nrow(df),
    y_obs = df$yObs,
    x = df$x
  )

  fit <- model$sample(data = dat, refresh = 0, show_messages = FALSE)
  fit$sampler_diagnostics()

  out <- fit$summary(
    variables = c("a", "b", "oddsratio"), median, quantile,
    .args = list(probs = c(0.025, .975), names = FALSE)
  )

  out <- list(
    point_a = out$median[out$variable == "a"],
    point_b = out$median[out$variable == "b"],
    point_or = out$median[out$variable == "oddsratio"],
    lower_a = out$quantile.1[out$variable == "a"],
    lower_b = out$quantile.1[out$variable == "b"],
    lower_or = out$quantile.1[out$variable == "oddsratio"],
    upper_a = out$quantile.2[out$variable == "a"],
    upper_b = out$quantile.2[out$variable == "b"],
    upper_or = out$quantile.2[out$variable == "oddsratio"]
  )

  return(out)
}

sim_method_mle <- function(df) {
  nll <- function(pars, data, se, sp) {
    # Parameters
    a <- pars[1]
    b <- pars[2]

    # Calculate p
    p <- (1 - sp) + (se + sp - 1) * plogis(a + b * data$x)

    # Calculate likelihood
    ll <- data$yObs * log(p) + (1 - data$yObs) * log(1 - p)

    # Return
    -sum(ll)
  }

  # We need to provide the point estimates for se and sp
  mle <- optim(
    par = c(a = -3, b = 0), fn = nll,
    data = df, se = 0.72, sp = 0.85,
    hessian = TRUE
  )
  std_error <- rlang::try_fetch( 
    # Use try in case that standard errors cannot be calculated
    {
      mle$hessian %>%
        solve() %>%
        diag() %>%
        sqrt()
    },
    error = function(cnd) {
      NA
    }
  )

  if (std_error %>% is.na() %>% all()) {
    out <- list(
      point_a = NA,
      point_b = NA,
      point_or = NA,
      lower_a = NA,
      lower_b = NA,
      lower_or = NA,
      upper_a = NA,
      upper_b = NA,
      upper_or = NA
    )
  } else {
    out <- list(
      point_a = mle$par["a"],
      point_b = mle$par["b"],
      point_or = exp(mle$par["b"]),
      lower_a = mle$par["a"] - 1.96 * std_error["a"],
      lower_b = mle$par["b"] - 1.96 * std_error["b"],
      lower_or = exp(mle$par["b"] - 1.96 * std_error["b"]),
      upper_a = mle$par["a"] + 1.96 * std_error["a"],
      upper_b = mle$par["b"] + 1.96 * std_error["b"],
      upper_or = exp(mle$par["b"] + 1.96 * std_error["b"])
    )
  }

  return(out)
}

# Extract interesting measures that vary across simulation iterations
sim_method_real <- function(df) {
  out <- list(
    p_yObs_x1 = mean(df$yObs[df$x == 1]),
    p_yObs_x0 = mean(df$yObs[df$x == 0]),
    p_yTrue_x1 = mean(df$yTrue[df$x == 1]),
    p_yTrue_x0 = mean(df$yTrue[df$x == 0]),
    a_true = qlogis(mean(df$yTrue[df$x == 0])),
    b_true = qlogis(mean(df$yTrue[df$x == 1])) - qlogis(mean(df$yTrue[df$x == 0])),
    or_true = exp(qlogis(mean(df$yTrue[df$x == 1])) - qlogis(mean(df$yTrue[df$x == 0]))),
    or_obs = (sum(df$yObs[df$x == 1] == 1) / sum(df$yObs[df$x == 1] == 0)) /
      (sum(df$yObs[df$x == 0] == 1) / sum(df$yObs[df$x == 0] == 0)),
    se_true = mean(df$yObs[df$yTrue == 1] == 1),
    sp_true = mean(df$yObs[df$yTrue == 0] == 0)
  )

  return(out)
}

# Evaluation function
sim_eval <- function(fit_results) {
  nrep <- fit_results$.rep %>%
    as.numeric() %>%
    max()

  ors <- c(0.8, 1.0, 1.2)
  real_vars <- c(
    "p_yObs_x1", "p_yObs_x0", "p_yTrue_x1", "p_yTrue_x0", "a_true", "b_true",
    "or_true", "or_obs", "se_true", "sp_true"
  )
  non_real_meth <- c("corr", "corr_priors", "mle", "naive", "true")

  for (i in 1:nrep) {
    for (j in seq_along(ors)) {
      for (k in seq_along(real_vars)) {
        fit_results[
          fit_results$.rep == as.character(i) &
            fit_results$ORx == ors[j] &
            fit_results$.method_name %in% non_real_meth,
          real_vars[k]
        ] <- fit_results[
          fit_results$.rep == as.character(i) &
            fit_results$ORx == ors[j] &
            fit_results$.method_name == "real",
          real_vars[k]
        ]
      }
    }
  }

  fit_results %<>% filter(.method_name != "real")

  fit_results %<>%
    mutate(
      contains_a_true = case_when(
        lower_a <= a_true & upper_a >= a_true ~ TRUE,
        lower_a > a_true ~ FALSE,
        upper_a < a_true ~ FALSE
      ),
      contains_b_true = case_when(
        lower_b <= b_true & upper_b >= b_true ~ TRUE,
        lower_b > b_true ~ FALSE,
        upper_b < b_true ~ FALSE
      ),
      contains_or_true = case_when(
        lower_or <= or_true & upper_or >= or_true ~ TRUE,
        lower_or > or_true ~ FALSE,
        upper_or < or_true ~ FALSE
      ),
      contains_or_obs = case_when(
        lower_or <= or_obs & upper_or >= or_obs ~ TRUE,
        lower_or > or_obs ~ FALSE,
        upper_or < or_obs ~ FALSE
      ),
      diff_a = point_a - a_true,
      diff_b = point_b - b_true,
      diff_or = point_or - or_true
    )

  return(fit_results)
}

      .method_name == "corr" ~ "C",
      .method_name == "corr_priors" ~ "P",
      .method_name == "mle" ~ "M"
    ),
    precision_or = upper_or - lower_or,
    diff_or_obs = or_obs - or_true,
    or_true_sign = case_when(
      or_true > 1 ~ ">1",
      or_true == 1 ~ "=1",
      or_true < 1 ~ "<1"
    ),
    diff_good = case_when(
      abs(diff_or_obs) > abs(diff_or) ~ "Better",
      abs(diff_or_obs) <= abs(diff_or) ~ "Worse"
    )
  )
